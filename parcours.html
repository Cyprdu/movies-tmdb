Prérequis

Avant de plonger dans les algorithmes de tri, voici les concepts Python que tu dois maîtriser. Les listes sont la structure de données principale que l'on va trier. Une liste Python est une séquence ordonnée et modifiable.

python 
# Créer et manipuler une liste
ma_liste = [64, 34, 25, 12, 22, 11, 90]

# Accès par index (commence à 0)
print(ma_liste[0])   # → 64
print(ma_liste[-1])  # → 90 (dernier élément)

# Longueur d'une liste
n = len(ma_liste)   # → 7

# Échanger deux éléments (indispensable pour les tris !)
ma_liste[0], ma_liste[1] = ma_liste[1], ma_liste[0]
print(ma_liste)  # → [34, 64, 25, 12, 22, 11, 90]

La boucle for avec range() est omniprésente dans les tris. La boucle while est utilisée pour reculer dans la liste.

python 
# Parcourir une liste avec for
for i in range(len(ma_liste)):
    print(f"Index {i} → {ma_liste[i]}")

# range(n) génère 0, 1, 2, ..., n-1
# range(1, n) génère 1, 2, ..., n-1
# range(n-1, -1, -1) génère n-1, ..., 1, 0 (en sens inverse)

# Boucle while (utile pour le tri par insertion)
j = 3
while j > 0 and ma_liste[j-1] > ma_liste[j]:
    ma_liste[j], ma_liste[j-1] = ma_liste[j-1], ma_liste[j]
    j -= 1
💡
Astuce clé — L'échange en Python Python permet d'échanger deux variables sans variable temporaire grâce au tuple unpacking : a, b = b, a — c'est ce que l'on utilise dans tous les algorithmes de tri !

Tri par Insertion

          
Comme trier des cartes dans sa main — on insère chaque carte à la bonne place

        

        O(n²)
      

      


        

          Le tri par insertion est le plus naturel des algorithmes de tri. Imagine que tu tiens des cartes à jouer : tu prends chaque nouvelle carte et tu la glisses à la bonne position parmi celles que tu tiens déjà.
        


        

          🎯
          

            Idée principale
            On maintient une partie gauche de la liste toujours triée. Pour chaque nouvel élément, on le décale vers la gauche jusqu'à trouver sa place.
          

        


        

          
Trace d'exécution — liste [5, 2, 4, 6, 1, 3]

          

            

              init
              

                524613
              

              Départ — [5] déjà "trié"
            

            

              i=1
              

                254613
              

              2 < 5 → on insère 2 avant 5
            

            

              i=2
              

                245613
              

              4 s'insère entre 2 et 5
            

            

              i=3
              

                245613
              

              6 > 5 → reste en place
            

            

              i=4
              

                124563
              

              1 remonte tout en tête
            

            

              fin
              

                123456
              

              Liste triée ✓
            

          

        


        

          Voici l'implémentation complète, étape par étape :
        


        

          

            

            python — tri_insertion.py
          

          
def tri_insertion(liste):
    """
    Tri par insertion — O(n²) en moyenne et au pire cas
    O(n) dans le meilleur cas (liste déjà triée)
    """
    # On commence à l'index 1 (l'élément 0 est trivalement "trié")
    for i in range(1, len(liste)):
        
        # L'élément à insérer à la bonne position
        cle = liste[i]
        
        # On recule dans la partie triée
        j = i - 1
        
        # Tant qu'on n'est pas au début ET que l'élément précédent est plus grand
        while j >= 0 and liste[j] > cle:
            liste[j + 1] = liste[j]  # Décaler l'élément vers la droite
            j -= 1
        
        # On place la clé à sa position correcte
        liste[j + 1] = cle
    
    return liste

# Test
ma_liste = [64, 34, 25, 12, 22, 11, 90]
print("Avant :", ma_liste)
tri_insertion(ma_liste)
print("Après :", ma_liste)

O(n²)

Complexité espace

            
O(1)

            
En place, pas de mémoire extra

          

          

            
Stable ?

            
✓ Oui

            
Préserve l'ordre des égaux

          

          

            
Ideal pour

            
Petites listes

            
ou presque triées

          

        


        

          ⚠️
          

            Piège classique !
            La variable cle = liste[i] est indispensable. Si tu fais directement liste[j+1] = liste[j] sans sauvegarder la clé, tu perdras la valeur originale !

Tri par Sélection

          
Cherche le minimum, place-le, recommence — simple et méthodique

        

        O(n²)
      

      


        

          Le tri par sélection est basé sur une idée simple : à chaque étape, on sélectionne le plus petit élément de la partie non triée et on le place à la bonne position.
        


        

          
Trace d'exécution — liste [64, 25, 12, 22, 11]

          

            

              init
              

                6425122211
              

              Aucun élément trié
            

            

              i=0
              

                1125122264
              

              Min=11 (idx 4) ↔ 64 (idx 0)
            

            

              i=1
              

                1112252264
              

              Min=12 (idx 2) ↔ 25 (idx 1)
            

            

              i=2
              

                1112222564
              

              Min=22 (idx 3) ↔ 25 (idx 2)
            

            

              fin
              

                1112222564
              

              Liste triée ✓
            

          

        


        

          

            

            python — tri_selection.py
          

          
def tri_selection(liste):
    """
    Tri par sélection — O(n²) dans tous les cas
    Toujours n*(n-1)/2 comparaisons, peu importe l'entrée
    """
    n = len(liste)
    
    for i in range(n):
        
        # Chercher l'index du minimum dans liste[i:]
        idx_min = i  # On suppose que le min est à l'index i
        
        for j in range(i + 1, n):
            if liste[j] < liste[idx_min]:
                idx_min = j  # Nouveau minimum trouvé !
        
        # Échanger le minimum trouvé avec l'élément à la position i
        liste[i], liste[idx_min] = liste[idx_min], liste[i]
    
    return liste

# Test
ma_liste = [64, 25, 12, 22, 11]
print(tri_selection(ma_liste))  # → [11, 12, 22, 25, 64]

        


        

          ✅
          

            Point fort du tri sélection
            Il effectue exactement n-1 échanges — c'est le minimum possible pour un tri par échanges. Utile quand les échanges sont coûteux (ex: écrire sur disque).

Tri Fusion (Merge Sort)

          
Divise pour mieux régner — la récursion au service du tri

        

        O(n log n)
      

      


        

          Le tri fusion utilise le paradigme Diviser pour Régner (Divide and Conquer). L'idée : diviser la liste en deux moitiés, trier chaque moitié récursivement, puis fusionner les deux moitiés triées. C'est un algorithme bien plus efficace que les deux précédents pour les grandes listes.
        


        

          🌳
          

            Visualise un arbre
            Le tri fusion crée un arbre de récursion. Chaque niveau divise par 2, donc la profondeur est log₂(n). À chaque niveau, on fait O(n) travail pour fusionner. Total : O(n × log n).
          

        


        

          
Arbre de récursion — [38, 27, 43, 3, 9, 82]

          

            

              Div.1
              

                382743
                |
                3982
              

              Divise en deux moitiés
            

            

              Div.2
              

                38
                |
                2743
                ||
                3
                |
                982
              

              Divise encore...
            

            

              Base
              

                382743
                ||
                3982
              

              Listes de taille 1 → triées
            

            

              Fus.1
              

                273843
                ||
                3982
              

              Fusion des sous-listes
            

            

              Fus.2
              

                3927384382
              

              Fusion finale ✓
            

          

        


        

          Le tri fusion a deux parties : la fonction principale qui divise récursivement, et la fonction de fusion (fusionner) qui combine deux listes triées.
        


        

          

            

            python — tri_fusion.py
          

          
def fusionner(gauche, droite):
    """Fusionne deux listes triées en une seule liste triée"""
    resultat = []
    i = j = 0
    
    # Tant qu'il reste des éléments dans les deux listes
    while i < len(gauche) and j < len(droite):
        if gauche[i] <= droite[j]:
            resultat.append(gauche[i])
            i += 1
        else:
            resultat.append(droite[j])
            j += 1
    
    # Ajouter les éléments restants (une seule liste aura des restes)
    resultat += gauche[i:]
    resultat += droite[j:]
    
    return resultat


def tri_fusion(liste):
    """
    Tri fusion récursif — O(n log n) dans tous les cas
    Utilise O(n) de mémoire supplémentaire
    """
    # Cas de base : liste de 0 ou 1 élément, déjà triée
    if len(liste) <= 1:
        return liste
    
    # Diviser en deux moitiés
    milieu = len(liste) // 2
    gauche = tri_fusion(liste[:milieu])   # Récursion gauche
    droite = tri_fusion(liste[milieu:])   # Récursion droite
    
    # Fusionner les deux moitiés triées
    return fusionner(gauche, droite)


# Test
ma_liste = [38, 27, 43, 3, 9, 82, 10]
print("Avant :", ma_liste)
ma_liste = tri_fusion(ma_liste)
print("Après :", ma_liste)

        


        

          ⚠️
          

            Différence importante !
            Contrairement aux tris précédents, tri_fusion retourne une nouvelle liste. Il ne modifie pas la liste originale en place. Tu dois donc faire : ma_liste = tri_fusion(ma_liste)
